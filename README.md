Проект содержит автоматические тесты для проверки API учебного сервиса Яндекс Самокат.
Документация API доступна по адресу: qa-scooter.praktikum-services.ru/docs/.

Тесты покрывают следующие ручки:

Создание курьера (POST /api/v1/courier)

Логин курьера (POST /api/v1/courier/login)

Создание заказа (POST /api/v1/orders)

Получение списка заказов (GET /api/v1/orders)

В ходе тестирования были обнаружены следующие баги:

1. Создание курьера без поля firstName
Ожидаемое поведение: при отсутствии обязательного поля (firstName) сервер должен вернуть ошибку 400.

Фактическое поведение: сервер возвращает статус 201 и успешно создаёт курьера.

Шаги воспроизведения:

Отправить POST-запрос на /api/v1/courier с телом, содержащим только login и password.
Проверить код ответа.
Приоритет: средний

2. Авторизация без обязательного поля
Ожидаемое поведение: при отсутствии поля login или password в запросе на авторизацию сервер должен вернуть ошибку 400.

Фактическое поведение: сервер возвращает статус 504 Gateway Timeout.

Шаги воспроизведения:

Отправить POST-запрос на /api/v1/courier/login с телом, содержащим только одно поле (например, только login).
Проверить код ответа.
Приоритет: высокий
